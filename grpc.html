Status codes

Errors use the standard gRPC status codes. A trailing grpc-status and grpc-message are returned on every call. Server-side details are attached as google.rpc.ErrorInfo when available.

# Code Used for
0 OK Successful response.
1 CANCELLED Caller cancelled the RPC.
3 INVALID_ARGUMENT Malformed field, e.g. invalid SemVer.
4 DEADLINE_EXCEEDED Caller deadline elapsed before completion.
5 NOT_FOUND No such package, version, or interface.
6 ALREADY_EXISTS Version already published; cannot overwrite.
7 PERMISSION_DENIED Token lacks the required scope.
8 RESOURCE_EXHAUSTED Per-token quota exceeded.
16 UNAUTHENTICATED Missing or invalid bearer token.
13 INTERNAL Unexpected server-side failure.
14 UNAVAILABLE Transient outage. Retry with backoff.
Guide

Deadlines & cancellation

Every call should set a deadline. Server defaults are conservative (30 s) and will be enforced even if the client omits one. Cancelled RPCs surface as CANCELLED on both sides; servers stop work immediately. Streaming RPCs may be cancelled mid-flight without leaking resources.

Health

grpc.health.v1.Health

Standard gRPC health-checking protocol. Use Check for one-shot probes and Watch for push-based health propagation.

UNARY grpc.health.v1.Health/Check

Returns SERVING when the registry is accepting traffic. Cheap; no datastore access.

Status
  • OK Service is up.
  • UNAVAIL Service is draining or restarting.
Example
Go Rust TypeScript Python Java 
import (
    "context"
    "time"

    healthpb "google.golang.org/grpc/health/grpc_health_v1"
)

func probe(client healthpb.HealthClient) error {
    ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
    defer cancel()

    resp, err := client.Check(ctx, &healthpb.HealthCheckRequest{
        Service: "registry.v1.Packages",
    })
    if err != nil { return err }
    if resp.GetStatus() != healthpb.HealthCheckResponse_SERVING {
        return fmt.Errorf("not serving: %v", resp.GetStatus())
    }
    return nil
}
SERVER grpc.health.v1.Health/Watch

Pushes a HealthCheckResponse on every state transition. The first message is sent immediately with the current state; subsequent messages are sent only on change. Closing the stream is the canonical way to stop watching.

Packages

registry.v1.Packages

Inspect packages, list versions, and manage publication. Repository paths are case-sensitive and slash-separated; the registry field is the upstream domain.

UNARY registry.v1.Packages/GetPackage

Resolve a single repository to its Package with the resolved latest tag.

Request
message GetPackageRequest {
  string registry   = 1; // "example.com"
  string repository = 2; // "hello-world"
}
Response
message GetPackageResponse {
  Package package = 1;
}
Status
  • OK Returns Package.
  • NOT_FOUND Repository is not indexed.
  • INVALID Empty registry or repository field.
Example
Go Rust TypeScript Python Java 
resp, err := client.GetPackage(ctx, &registrypb.GetPackageRequest{
    Registry:   "example.com",
    Repository: "hello-world",
})
if err != nil {
    if status.Code(err) == codes.NotFound {
        return nil, ErrUnknownPackage
    }
    return nil, err
}
fmt.Println(resp.GetPackage().GetLatest()) // "0.1.0"
SERVER registry.v1.Packages/ListVersions

Server-streams every published version for a repository, newest first. The stream completes naturally when the catalog is exhausted; there is no explicit pagination cursor.

Request
message ListVersionsRequest {
  string registry       = 1;
  string repository     = 2;
  bool   include_yanked = 3;
}
Response (stream)
message ListVersionsResponse {
  PackageVersion version = 1;
}
UNARY registry.v1.Packages/GetVersion

Resolve a single tag to its PackageVersion. Accepts a SemVer tag or the literal latest.

Request
message GetVersionRequest {
  string registry   = 1;
  string repository = 2;
  string version    = 3;
}
Response
message GetVersionResponse {
  PackageVersion version = 1;
}
CLIENT registry.v1.Packages/Publish Auth required

Client-streams a new version. The first message is the manifest header; subsequent messages stream the wasm component as PublishChunk frames. The server returns a single response when the stream is closed.

Request (stream)
message PublishRequest {
  oneof payload {
    PublishHeader header = 1;
    PublishChunk  chunk  = 2;
  }
}
Response
message PublishResponse {
  PackageVersion version = 1;
}
Status
  • OK Version accepted; canonical PackageVersion returned.
  • EXISTS Tag already published.
  • UNAUTH Token missing or invalid.
UNARY registry.v1.Packages/YankVersion Auth required

Mark a version as yanked. Yanked versions remain resolvable by exact digest but are excluded from ListVersions unless include_yanked is set. Reversible by re-publishing the same digest.

Search

registry.v1.Search

Locate packages by the WIT interfaces their worlds import or export, and walk the resolved dependency graph interactively.

SERVER registry.v1.Search/SearchByInterface

Server-streams matching Package entries whose worlds import or export the given interface.

Request
message SearchByInterfaceRequest {
  string      interface = 1;
  SearchKind  kind      = 2;
}

enum SearchKind {
  KIND_UNSPECIFIED = 0;
  KIND_IMPORT      = 1;
  KIND_EXPORT      = 2;
}
Response (stream)
message SearchByInterfaceResponse {
  Package package = 1;
}
BIDI registry.v1.Search/ResolveGraph

Bidirectional dependency resolver. The client sends ResolveRequest messages and the server replies with ResolveResponse messages as candidates are discovered. Suitable for incremental lockfile generation.

Messages

Object reference

Canonical Protobuf shapes used across the service. Each row is prefixed with the Protobuf field number — these are stable and will never be reassigned.

Package

A registry-qualified repository pointer with a resolved latest tag.

1 registry string Upstream host name.
2 repository string Slash-separated repository path.
3 latest string Highest non-yanked SemVer tag, empty if none.

PackageVersion

A single tag with manifest metadata and dependency graph.

1 version string · semver SemVer tag.
2 digest string · sha256 OCI manifest digest.
3 size int64 Total wasm layer size in bytes.
4 yanked bool True if the version was withdrawn.
5 published_at google.protobuf.Timestamp Server-assigned publish time.
6 imports repeated string WIT interfaces imported by the world.
7 exports repeated string WIT interfaces exported by the world.

PublishChunk

One frame of the wasm component during a Publish stream. Chunks are at most 1 MiB; the final chunk has last = true.

1 data bytes Raw bytes from the wasm component.
2 last bool Set on the final chunk.

ResolveRequest

A single resolution query within a ResolveGraph stream.

1 registry string Upstream host name.
2 repository string Repository path.
3 version_req string · SemVer range SemVer range, e.g. ^0.2.

ResolveResponse

A resolved candidate for a single ResolveRequest.

1 request_index uint32 Index of the originating request in the stream.
2 resolved PackageVersion The selected version, or empty if no match.
