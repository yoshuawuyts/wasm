Crate wasm_component §

1.0.0 source

A minimal, ergonomic runtime for WebAssembly Components. Provides a typed Linker, host bindings, and a wire-compatible ABI for interface types. Designed to pair cleanly with wasmtime while remaining embeddable in smaller hosts.

Components are loaded from a binary artifact, linked against host-provided imports, and then instantiated into a Store. Each instantiation produces a strongly-typed Instance whose exported functions can be called directly from Rust.

examples/hello.rs 
// Load, link, and run a component end-to-end.
use wasm_component::Component;
use wasm_component::{Engine, Linker, Store};

fn main() -> wasm_component::Result<()> {
    let engine    = Engine::default();
    let component = Component::from_file(&engine, "hello.wasm")?;

    let mut linker = Linker::<()>::new(&engine);
    wasm_component::wasi::add_to_linker(&mut linker)?;

    let mut store = Store::new(&engine, ());
    let instance  = linker.instantiate(&mut store, &component)?;

    instance.get_typed_func::<(), ()>(&mut store, "run")?
            .call(&mut store, ())?;
    Ok(())
}

S Struct wasm_component::Component §

1.0.0 source 
pub struct Component { /* private fields */ }

A compiled WebAssembly Component. Component values are immutable and cheaply cloneable — internally they share an Arc-ed compiled artifact, so passing one across threads is essentially free.

Compilation is performed by an Engine, typically via Component::from_file or Component::from_bytes. Both methods validate the binary against the component-model spec before producing a Component.

See also
  • Engine — the compilation environment that produces components.
  • Linker — supplies imports for a component at link time.
  • validate — check a binary without compiling it.

pub fn from_file<P: AsRef<Path>>(
    engine: &Engine,
    path: P,
) -> Result<Self>
§ source
since 1.0.0

Load and compile a component from a file on disk. The file is memory-mapped where supported, so large components do not need to be fully read into memory before compilation begins.

Parameters
engine
The Engine to compile with. Compilation caches are keyed per-engine.
path
Path to a .wasm component binary. May be absolute or relative to the current working directory.
Errors
  • Error::Io — the file could not be opened or read.
  • Error::Validation — the binary is not a valid component (see validate for a non-compiling check).
  • Error::Compilation — the component is well-formed but could not be compiled for the current target.
Example
Compile a component from disk 
let engine    = Engine::default();
let component = Component::from_file(&engine, "./dist/app.wasm")?;
println!("exports: {}", component.exports().count());

Compile a component directly from an in-memory byte slice. Prefer this when the binary is embedded in the host executable (include_bytes!) or delivered over the network.

Parameters

The Engine to compile with.
bytes
The raw bytes of a component binary. The slice need not outlive the returned Component.
Errors

  • Error::Compilation — compilation failed for the current target.
